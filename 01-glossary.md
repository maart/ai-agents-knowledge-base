# Глоссарий

Рабочие определения для проекта. Они будут уточняться по мере чтения источников.

## Agent

Система, в которой модель выбирает или инициирует действия во внешней среде, получает и анализирует результаты этих действий (observation), а затем продолжает работу в несколько шагов для достижения цели.

## Agentic system

Более широкий термин для систем, где есть агентские свойства: использование инструментов, планирование, состояние, автономные действия. Может включать как fully autonomous [agents](notes/concepts/agent.md), так и более контролируемые [workflows](notes/concepts/workflow.md).

## Autonomy

Степень свободы системы самостоятельно выбирать следующие действия, инструменты, порядок работы и момент остановки.

## Chatbot

Диалоговый интерфейс, который отвечает пользователю. Chatbot может использовать инструменты, но сам по себе термин не означает агентскую архитектуру.

## Workflow

Предопределенный процесс с шагами, условиями и правилами. LLM может быть компонентом workflow, но не обязательно управляет всем процессом.

## Tool use

Способность модели вызывать внешние функции, API, браузер, поиск, кодовую среду, базы данных или другие системы.

## Model/API layer

Базовый слой агентской системы: модель и API, через которые приложение получает reasoning, generation, tool calling, multimodal capabilities и доступ к контексту. Этот слой дает агенту "умение думать и отвечать", но сам по себе еще не делает систему агентом: агентность появляется, когда модель встроена в цикл действий, состояния, инструментов, контроля и оценки.

## SDK and orchestration

Слой разработки и управления, где агентская система собирается в коде: agents, tools, guardrails, handoffs, sessions, state, tracing, retries и execution flow. SDK дает готовые primitives, а orchestration определяет, как шаги соединяются, кто выбирает следующий шаг, где хранится state и как система восстанавливается после ошибок.

## Visual builders and agent platforms

Продуктово-платформенный слой для сборки агентских workflows без полного ручного кодирования: visual builders, connectors, embedded chat, versioning, deployment и evals. Такие platforms не заменяют model/API layer или SDK, а собирают их в управляемую product experience для проектирования, тестирования и эксплуатации agents.

## Function calling

Механизм, при котором модель возвращает структурированный запрос на вызов функции, а приложение выполняет этот вызов и возвращает результат модели.

## RAG

Retrieval-Augmented Generation: подход, при котором система ищет релевантные документы или фрагменты данных и добавляет их в контекст модели перед генерацией ответа.

## Memory

Состояние или знания, сохраняемые между шагами или сессиями. Может быть краткосрочной памятью текущей задачи, долговременной памятью пользователя или внешним хранилищем знаний.

## Planning

Декомпозиция цели на шаги и выбор порядка действий. План может быть явным текстовым артефактом или неявной частью рассуждения модели.

## Observation

Observation, или наблюдение результата, — это получение и анализ обратной связи после действия агента. В агентском цикле `observe -> decide/think -> act -> observe` этот термин связывает систему с внешней средой: результатом tool call, изменением страницы, выводом программы, ответом API или другим сигналом о том, что произошло после действия.

## Orchestration

Слой управления агентской системой: маршрутизация, вызовы инструментов, handoffs, состояние, ограничения, повторные попытки, остановка и наблюдаемость.

## Handoff

Передача задачи другому агенту, модели, workflow или человеку.

## Guardrails

Ограничения и проверки, которые снижают риск нежелательных действий: валидация входов/выходов, разрешения на инструменты, политики безопасности, human approval.

## Evals

Методы оценки качества системы: тестовые наборы, сценарии, автоматические проверки, human review, regression tests, измерение успешности выполнения задачи.

## MCP

Model Context Protocol: открытый протокол для подключения AI-приложений к инструментам, ресурсам и внешним источникам контекста через стандартизированный интерфейс.

## Computer use

Способность агента взаимодействовать с графическим интерфейсом компьютера или браузера: видеть экран, кликать, вводить текст, читать результат.

## Long-running agent

Агент, который выполняет задачу дольше одного короткого запроса: сохраняет состояние, возобновляет работу, переносит контекст и требует надежного контроля выполнения.

## Multi-agent system

Система из нескольких агентов с разными ролями или специализациями, которые совместно решают задачу.
