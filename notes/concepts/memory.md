# Memory

Тип: собственный синтез.

## Основная идея

В агентских системах "memory" может означать несколько разных вещей. Их важно не смешивать.

## Виды памяти

- Context window: информация, прямо переданная модели сейчас.
- Session state: состояние текущей задачи между шагами.
- Retrieval/RAG: поиск релевантных фрагментов во внешнем хранилище.
- Long-term memory: данные, сохраняемые между сессиями.
- Skill library: сохраненные процедуры или код, которые агент может повторно использовать.
- Audit trail: история действий и решений для проверки.

## Почему это важно

Ошибки часто возникают не потому, что "у агента плохая память", а потому что неясно, какой вид памяти нужен для задачи.

## Практическое правило

Для v1 agentic system лучше начинать с явного session state и audit trail. Long-term memory добавлять только тогда, когда понятно, что именно нужно помнить и как это проверять.
