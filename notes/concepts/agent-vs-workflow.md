# Agent vs workflow

Тип: собственный синтез.

## Основная идея

[Workflow](workflow.md) заранее задает маршрут выполнения. [Agent](agent.md) выбирает часть маршрута динамически, исходя из цели, контекста и observations, то есть наблюдений результатов действий или сигналов среды.

Для отдельных вводных см. [Agent](agent.md) и [Workflow](workflow.md).

Это не бинарная оппозиция. Большинство практичных систем находятся между двумя полюсами:

- workflow задает рамки, права, этапы и остановку;
- agent выбирает действия внутри этих рамок.

## Chatbot

Chatbot — это интерфейс общения. Он может быть частью агентской системы, но сам по себе не означает агентность.

## Workflow

Workflow полезен, когда задача повторяемая, риски высокие и путь известен заранее. Хороший workflow легче тестировать, воспроизводить и объяснять.

## Agent

Agent полезен, когда путь заранее неизвестен: нужно исследовать, пробовать инструменты, адаптироваться к ошибкам и менять план по ходу работы.

## Рабочий критерий

Если модель только заполняет один шаг в заранее заданном процессе — это LLM workflow. Если модель выбирает следующий шаг и взаимодействует со средой — это agentic system.

Ветвления сами по себе не делают систему агентской. Например, n8n-сценарий:

```text
если ссылка есть -> прочитать ресурс
если ссылки нет -> идти дальше без дополнительной информации
```

остается workflow, если эти ветки заранее описаны человеком. Даже если один из шагов вызывает LLM, модель здесь выполняет роль внутри маршрута, а не управляет маршрутом.

Система становится ближе к agentic workflow, если модель сама решает, какой значимый шаг сделать дальше:

```text
оценить, достаточно ли информации;
решить, читать ли ссылку;
выбрать, искать ли дополнительный источник;
выбрать инструмент;
вернуться к уточнению;
завершить задачу.
```

Короткая формула:

- predefined branching -> workflow;
- model-selected next action inside fixed boundaries -> agentic workflow;
- model-led observe/decide/act loop -> agentic system.
