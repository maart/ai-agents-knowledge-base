# Tool use

Тип: собственный синтез.

## Основная идея

Tool use дает модели возможность выходить за пределы текста: искать, считать, читать файлы, вызывать API, управлять браузером, запускать код или обращаться к внешним системам.

## Зачем это нужно

Без инструментов модель ограничена знаниями в параметрах и текущим контекстом. С инструментами она может:

- получать актуальную информацию;
- выполнять точные вычисления;
- работать с приватными данными;
- изменять состояние внешних систем;
- проверять свои гипотезы через выполнение.

## Важное различие

Function calling — механизм структурированного вызова функции.

Tool use — более широкая способность системы использовать внешние возможности. Она включает function calling, built-in tools, MCP, browser use и computer use.

## Риск

Tool use увеличивает полезность, но также расширяет поверхность риска. Нужно проектировать разрешения, журналирование, подтверждения и ограничения на действия.
