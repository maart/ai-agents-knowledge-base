# Planning

Тип: собственный синтез.

## Основная идея

Planning — это способность разбить цель на шаги и менять порядок действий по мере появления новых наблюдений.

## Варианты планирования

- Неявное: модель выбирает следующий шаг без отдельного артефакта плана.
- Явное: модель или система создает список шагов.
- Graph-based: возможные переходы заданы как граф.
- Hierarchical: задача делится на подзадачи, роли или этапы.

## Важная оговорка

Больше планирования не всегда лучше. Для простых задач план может быть лишней задержкой. Для длинных задач план нужен, но должен обновляться после наблюдений.

## Что проверять

- План связан с целью или просто выглядит убедительно?
- Система замечает, когда план устарел?
- Есть ли правила остановки?
- Может ли человек понять, что агент собирается делать?
