# Long-term memory

Тип: собственный синтез.

## Основная идея

Long-term memory — это информация, которая сохраняется между сессиями и может влиять на будущие задачи агента.

## Что может храниться

В зависимости от продукта long-term memory может включать:

- устойчивые предпочтения пользователя;
- сведения о проектах;
- рабочие правила команды;
- прошлые решения;
- часто используемые источники;
- навыки или процедуры;
- summary предыдущих задач.

## Чем это отличается от session state

[Session state](session-state.md) нужен для текущей задачи. Long-term memory переживает задачу и возвращается позже.

Это делает long-term memory мощной, но рискованной: ошибка, попавшая в долговременную память, может влиять на много будущих действий.

## Когда добавлять

Long-term memory полезна, если:

- агент работает с одним пользователем или командой долго;
- задачи повторяются;
- есть устойчивые предпочтения или контекст проекта;
- пользователь ожидает персонализации;
- прошлые решения действительно нужны в будущем.

## Риски

- Агент запоминает неправильный факт.
- Пользователь не понимает, что именно сохранено.
- Память нарушает приватность.
- Старые предпочтения становятся неактуальными.
- Система переиспользует память без проверки.

## Практическое правило

Long-term memory должна быть видимой, редактируемой и проверяемой. Лучше хранить меньше, но с ясным источником и причиной, чем автоматически запоминать все подряд.
