# Context window

Тип: собственный синтез.

## Основная идея

Context window — это объем информации, который модель может учитывать в одном запросе или одном шаге выполнения. Это не долговременная память, а текущий рабочий контекст, переданный модели прямо сейчас.

## Что туда попадает

В context window могут входить:

- системные инструкции;
- запрос пользователя;
- история диалога;
- найденные документы;
- результаты tool calls;
- промежуточный план;
- ограничения задачи;
- фрагменты кода, таблиц или файлов.

## Почему это не память в полном смысле

Context window исчезает или меняется между запросами, если приложение специально не сохраняет и не возвращает нужные данные. Модель не "помнит" прошлую информацию сама по себе; приложение решает, что снова передать в контекст.

## Роль в agentic systems

Для агента context window — это рабочий стол. Если в него не попали цель, ограничения, последние observations и важные результаты инструментов, агент может принять плохое решение.

## Риски

- Context overload: слишком много нерелевантного текста снижает качество.
- Missing context: важный факт не передан модели.
- Stale context: агент использует устаревшую информацию.
- Prompt injection: внешний текст внутри контекста пытается изменить поведение агента.

## Практическое правило

Context window нужно проектировать как входной интерфейс модели: передавать достаточно информации для решения шага, но не превращать контекст в бесконтрольную свалку.
