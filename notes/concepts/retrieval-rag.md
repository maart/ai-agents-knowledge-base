# Retrieval / RAG

Тип: собственный синтез.

## Основная идея

Retrieval-Augmented Generation (RAG) — это подход, при котором система ищет релевантные документы, фрагменты или записи во внешнем хранилище и добавляет их в context window перед генерацией ответа или следующим агентским шагом.

## Зачем это агентам

RAG позволяет агенту работать с информацией, которая не содержится в весах модели:

- внутренние документы;
- база знаний;
- проектные заметки;
- API-документация;
- история задач;
- корпоративные политики;
- свежие или приватные данные.

## Retrieval не равен memory

Retrieval — это механизм поиска. Memory — более широкое понятие. RAG может быть способом реализовать часть памяти, но не заменяет session state, audit trail или long-term memory.

## Где RAG особенно полезен

- Research agents.
- Support agents.
- Documentation agents.
- Coding agents, которые ищут релевантные файлы или docs.
- Internal knowledge assistants.

## Риски

- Найдены нерелевантные документы.
- В контекст попадает устаревшая информация.
- Агент принимает retrieved text за истину без проверки.
- Prompt injection попадает через внешние документы.
- Нет provenance: непонятно, откуда взят факт.

## Практическое правило

RAG должен возвращать не только текст, но и источник, дату, контекст и степень доверия. Для агентских систем retrieval лучше рассматривать как tool с observability, а не как скрытую магию.
