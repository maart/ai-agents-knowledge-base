# Skill library

Тип: собственный синтез.

## Основная идея

Skill library — это набор сохраненных процедур, инструментальных шаблонов, кода или повторяемых действий, которые агент может использовать в будущих задачах.

Это особый вид памяти: система запоминает не только факты, но и способы действия.

## Примеры

Skill library может содержать:

- готовые code snippets;
- workflow-шаблоны;
- инструкции для типовых задач;
- reusable prompts;
- проверенные команды;
- small scripts;
- процедуры работы с конкретным API;
- способы исправления частых ошибок.

## Почему это важно

Skill library позволяет агенту накапливать операционный опыт. Вместо того чтобы каждый раз решать задачу с нуля, агент может переиспользовать проверенные действия.

Исследовательский пример — Voyager: агент в Minecraft накапливает библиотеку исполняемых навыков и применяет их в новых ситуациях.

## Отличие от long-term memory

Long-term memory чаще хранит факты и предпочтения. Skill library хранит процедуры и способы действия.

## Риски

- Скилл устарел, но агент продолжает его применять.
- Скилл был создан для другого контекста.
- Скилл содержит небезопасную команду.
- Агент выбирает знакомый скилл вместо более подходящего решения.

## Практическое правило

Skill library должна иметь versioning, описание условий применения и проверку результата. Хороший skill — это не просто сохраненный текст, а воспроизводимое действие с понятными границами.
