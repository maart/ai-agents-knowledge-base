# Protocols and interoperability

Тип: собственный синтез.

## Основная идея

Agent ecosystem быстро движется от отдельных приложений к сетям инструментов, серверов, workflow и агентов. Поэтому важны протоколы: они задают общие способы подключения возможностей и обмена задачами между системами.

## MCP и A2A

MCP (Model Context Protocol) и A2A (Agent2Agent) решают разные задачи.

MCP отвечает на вопрос:

```text
как AI-приложению подключаться к tools, resources и prompts?
```

A2A отвечает на другой вопрос:

```text
как один агент может общаться, координироваться или обмениваться задачами с другим агентом?
```

Коротко:

- MCP: agent/application -> tools and context.
- A2A: agent -> agent.

## Почему это важно

Без протоколов каждый agent product строит собственные интеграции. Это быстро создает фрагментацию: разные tool schemas, разные способы авторизации, разные форматы результатов, разные модели capability discovery.

Протоколы обещают:

- повторно использовать integrations;
- подключать инструменты и агентов из разных систем;
- описывать capabilities в машинно-читаемом виде;
- стандартизировать transport, messages and artifacts;
- сделать audit и security более системными.

## Где появляется риск

Interoperability расширяет область действия агента. Чем легче подключить внешний инструмент или другого агента, тем важнее:

- identity: кто именно вызывает действие;
- permissions: какие действия разрешены;
- trust boundaries: чему можно доверять;
- provenance: откуда пришел artifact или результат;
- logging: как восстановить ход взаимодействия;
- sandboxing: где выполняется потенциально опасное действие.

## Связь с workflow

Протоколы не отменяют workflow. Наоборот, зрелые системы часто используют workflow как контрольный слой:

- workflow решает, когда можно подключить внешний tool или agent;
- MCP предоставляет стандартный доступ к capabilities;
- A2A позволяет делегировать задачу другому агенту;
- evals и audit trail проверяют, что произошло.

## Практическое правило

MCP стоит изучать как инфраструктуру tool/context integration. A2A стоит изучать как инфраструктуру межагентного взаимодействия. Оба важны для агентской экосистемы, но ни один из них сам по себе не делает систему надежной.
