# Multi-agent systems

Тип: собственный синтез.

## Основная идея

Multi-agent system использует несколько агентов или ролей для решения задачи. Частый паттерн: planner, researcher, executor, critic, reviewer.

## Когда это полезно

- Задача естественно делится на разные компетенции.
- Нужен независимый review.
- Есть параллельные ветки исследования.
- Нужно разделить полномочия и инструменты.

## Когда это вредно

- Несколько агентов повторяют одну и ту же работу.
- Координация дороже результата.
- Ошибки одного агента усиливаются другими.
- Разговор между агентами выглядит разумно, но не улучшает результат.

## Рабочий критерий

Multi-agent подход должен доказывать пользу через качество, скорость, надежность или управляемость. Если пользы нет, лучше один агент с ясным workflow.
